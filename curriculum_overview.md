# Computer Science Curriculum - Overview

## Semester_01 - Basics Computer Science

- Modul_00_Hardware_&_Binary_Basics
    - Understanding CPU, RAM (Stack vs. Heap), Binary System
- Modul_01_Computer_Logic_&_IPO_Model
    - Input-Process-Output (IPO) cycle and CIL/JIT compilation process
- Modul_02_Data_Types_&_Variables
    - Memory allocation for types, Operators, importance of "TryParse"
- Modul_03_Conditions_&_Logic
    - Control flow with if/else, Boolean logic, Guard Clauses
- Modul_04_Loops
    - Iteration patterns, Scope management, avoiding Off-by-One errors
- Modul_05_Methods_&_Functions
    - Pure functions, Side Effects, Method Overloading, Error Handling Patterns
- Modul_06_Professional_Debugging
    - Workflow for bug reproduction, Hypotheses, utilizing Breakpoints
- Modul_07_Collections_Light
    - Lists vs. Arrays, Memory efficiency, String Manipulation
- Modul_08_Strings_Parsing_&_Tokenizer
    - Building a robust Command-Parser using State Machines and StringBuilder

## Semester_02 - Advanced System Architecture

- Modul_01_Classes_&_Objects_Basics
    - Transitioning to Object-Oriented Programming (OOP), reference types, instance management
- Modul_02_Encapsulation_&_Constructors
    - Securing data integrity with "private set", protecting invariants, "valid-by-birth" objects
- Modul_03_Advanced_Lists_&_Data_Models
    - Working with complex objects, reference handling in collections, atomic operations
- Modul_04_Dictionary_&_Lookup_Thinking
    - Optimizing performance with O(1) lookups, implementing Registries, "Source of Truth" pattern
- Modul_05_Deep_Dive_Strings_&_Parsing_Patterns
    - Refining the parsing engine with advanced tokenization and production-grade "Try-Pattern"
- Modul_06_Error_Handling_&_Validation_Frameworks
    - Implementing the Result-Pattern, distinguishing Exceptions (Invariants) from Validation (Expected Errors), Error Propagation
- Modul_07_Enums_&_Type_Safety
    - Utilizing Enums for Directions and GameStates, eliminating "Magic Strings", Enum.TryParse normalization
- Modul_08_Object_Representation_&_Logging
    - Overriding ToString() for debugging, creating "Debug Views" for Player/Inventory, selective Logging patterns
- Modul_09_Architecture_Light_&_Separation_of_Concerns
    - Decoupling Logic from I/O, "Compute vs. Apply" principle, introduction to Service-oriented layers
- Modul_10_Mini-Project_01_Multi-Class_Systems
    - Building a 3-5 class system (Room, Item, Player, Command), focus on clean interaction and collection management
- Modul_11_Capstone_Project_Semester_02
    - Constructing a complex 5-8 class RPG Core Loop, world state management, and architectural scalability
